# First-Run Heuristics

Upgrade severity when a first-run problem:
- happens before the first success moment
- leaves the user unsure whether setup worked
- requires knowledge the product never taught
- makes the app appear dead or idle
- sends the user into system settings without a clear return path
- permanently poisons the user’s mental model after one unlucky failure

Common first-run killers:
- too many choices before any value appears
- permission prompts with weak pre-explanation
- subtle menu bar state changes that normal users never notice
- no test action, preview, or success proof after setup
- skipped onboarding paths that strand the user in a weird half-configured state
