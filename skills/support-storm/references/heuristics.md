# Support Inbox Heuristics

Good simulations include:
- at least one polite but confused user
- at least one angry user who is directionally right
- at least one user with the wrong diagnosis
- at least one public review that oversimplifies many bugs into one story
- at least one issue that support cannot solve in one reply

Warning signs:
- “it’s not working” without channel/version/provider details
- user language that signals trust collapse: flaky, creepy, scammy, not worth it
- repeated misunderstandings that docs alone will not fix
- failures where support needs hidden internal state the app never exposes
